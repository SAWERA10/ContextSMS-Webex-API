Authentication

Authentication APIs for login and token refresh

Auth - Login API

This is the starting point for any interaction with Sabrhub's protected services. When a user wants to access these services, they must first authenticate themselves by providing their credentials (username and password) through this API. On successful authentication, the API responds with an accessToken, a refreshToken, and other user-related information. The accessToken is a time-limited token that must be included in the header of subsequent API requests to prove the user's identity and authorization. This mechanism prevents unauthorized access and ensures that each request is made by a recognized and validated user. Additionally, the refreshToken is provided for use when the accessToken expires.

Once authenticated, the accessToken must be used in all protected API calls as a Bearer token in the Authorization header.

Auth - Access Token API

This API is used when the accessToken obtained from the Login API is close to its expiration. The expiration time is always provided in the Login API response. To maintain continuous access without needing to re-enter credentials, the refreshToken is used with this API to obtain a new accessToken. This process enhances security by regularly updating access tokens and ensures that user sessions remain secure without compromising convenience. Regularly refreshing tokens is a best practice in API security, as it limits the window of opportunity for any compromised token to be exploited.

Enterprise

Endpoints for creating, reading, and deleting enterprise accounts

Create Enterprise API

This endpoint allows for the creation of a new enterprise entity under a specific CSP. It accepts essential enterprise details such as name, contact information, and configuration flags such as defaultMmsSelection.

‼️ Note: If the MMS feature is set to Strict or Disabled, there's no need to send this parameter. It will be set according to your MMS feature. If it's Strict, then set to true; if it's Disabled, set to false

‼️ Note: If the MMS feature is set to Partial, you can pass either defaultMmsSelection=true or defaultMmsSelection=false.
